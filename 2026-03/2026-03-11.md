## 用語ミニ解説

> 本レポートで使用する専門用語を冒頭で簡単に説明します。はじめてAI市場レポートを読む方はここを先に確認してください。

- **LLM（Large Language Model）**: 大規模言語モデル。大量のテキストで学習したAIで、文章の生成・要約・翻訳などができます（例: GPT-5、Claude、Gemini）。
- **AIエージェント**: 人間が細かく指示しなくても、自律的に複数のステップのタスクをこなせるAIシステムです。
- **MCP（Model Context Protocol）**: AIモデルが外部ツールやデータベースに接続するための標準規格。AIのための「共通コンセント」のようなものです。
- **PR（Pull Request）**: コードの変更提案。GitHubなどで「このコードをメインブランチに取り込んでほしい」と依頼する仕組みです。コードレビューはこのPRを人間が確認して品質を担保する工程です。
- **マルチエージェント**: 複数のAIエージェントが並列・協調して動作するシステム。単一エージェントより複雑なタスクを高速にこなせます。
- **リサーチプレビュー**: 本番リリース前の試験的公開。限定ユーザーに提供されるベータ版の一種です。

### Claude Code Review — AnthropicがマルチエージェントPRレビューシステムを発表、AIコード急増時代の品質ボトルネックに正面から挑む

- **カテゴリ**: 新機能ローンチ
- **AI領域**: コーディングAI / AIエージェント・自動化
- **概要**: Anthropicは2026年3月9〜10日にかけて、「Claude Code Review」をTeam・Enterpriseプランのリサーチプレビューとして公開した。PRがオープンされると複数のAIエージェントが自動で起動し、バグ発見・検証・重要度ランク付けを並列で実行する。レビュー所要時間は平均約20分。Anthropicによると、同機能導入前は実質的なレビューコメントが付くPRは全体の16%に過ぎず、エンジニア1人あたりのコード出力が過去1年で200%増加した結果、ヒューマンレビューがボトルネック化していたという。料金はトークン量に応じた従量課金で、1レビューあたり平均15〜25ドル。なお、PRの最終マージ承認は必ず人間が行う仕様とされている。
- **ユーザー反応**: ポジティブ。X・Redditでは「AIが書いたコードをAIがレビューするループが完成した」という感想が多数。一方で「1PRあたり$15〜25は中規模チームには割高」「誤検知率が低いとはいえゼロではない」という懸念も見られる。企業ユーザーからは「PRが増えすぎてレビューが追いつかない問題を解決してくれる」との期待の声が目立つ。
- **情報の確度**: 公式発表
