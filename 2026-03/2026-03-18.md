## 用語ミニ解説

> 本レポートで使用する専門用語を冒頭で簡単に説明します。はじめてAI市場レポートを読む方はここを先に確認してください。

- **LLM（Large Language Model）**: 大規模言語モデル。大量のテキストで学習したAIで、文章の生成・要約・翻訳などができます（例: GPT-5、Claude、Gemini）。
- **AIエージェント**: 人間が細かく指示しなくても、自律的に複数のステップのタスクをこなせるAIシステムです。
- **MCP（Model Context Protocol）**: AIモデルが外部ツールやデータベースに接続するための標準規格。AIのための「共通コンセント」のようなものです。
- **コンテキストウィンドウ**: LLMが一度に処理できるテキストの量（トークン数）。大きいほど長い文書や会話を扱えます。
- **転換社債**: 一定の条件で株式に転換できる社債（借入証書）。スタートアップ・成長企業がまとまった資金を調達する際によく使われます。
- **LPU（Language Processing Unit）**: Groqが開発した推論専用チップ。GPUとは異なりLLMのトークン生成に特化した設計で、超高速推論が可能です。

### Claude Code — Opus 4.6 の出力トークン上限を 64k / 128k に大幅拡張

- **カテゴリ**: メジャーアップデート
- **AI領域**: コーディングAI
- **概要**: 2026年3月17日、Anthropicは Claude Code のアップデートを配信し、Claude Opus 4.6（Anthropicの最上位コーディングAIモデル）のデフォルト最大出力トークン数を従来比で引き上げ、デフォルト上限を64,000トークン、絶対上限を128,000トークンに設定した。Claude Sonnet 4.6 の絶対上限も同じく128kトークンへ統一。これにより、大規模なファイル書き換え・長大なPR差分生成・マルチファイルにまたがる長いコーディングセッションが、より少ない中断でこなせるようになる。同アップデートでは `/copy N`（N番目に新しいアシスタント返答をコピーするコマンド）の追加、サンドボックス内でのファイル読み取り権限を細かく制御できる `allowRead` 設定の追加、パーミッション処理・自動アップデート・会話履歴・コスト追跡に関する複数のバグ修正も含まれる。
- **ユーザー反応**: ポジティブ。「128kアウトプットはコードベース全体のリファクタリングを一発でやれるレベル」「長大なコード生成が途中で止まらなくなる」という歓迎コメントが多い。一部では「Opus 4.6でこの上限を使うとコスト爆発しないか」という懸念も見られる。
- **情報の確度**: 公式発表
- **ソース**: [Changelog - Claude Code Docs](https://code.claude.com/docs/en/changelog)（投稿日: 2026-03-17、ソース: Anthropic公式）、[Claude Code by Anthropic - Release Notes - March 2026 | Releasebot](https://releasebot.io/updates/anthropic/claude-code)（投稿日: 2026-03-17、ソース: Releasebot）

### Claude AI — 大規模サービス障害が発生、Claude Code・API 開発者にも影響（同日解消）

- **カテゴリ**: メジャーアップデート（障害・サービス品質）
- **AI領域**: LLM / チャットAI
- **概要**: 2026年3月17日、Claude AI（Anthropic の大規模言語モデルサービス）が複数の国・プラットフォームにわたって広範な障害を起こした。障害追跡プラットフォーム Downdetector には午後1時03分（米国太平洋時間）時点で6,800件超の障害報告が集中。Claude.ai のチャットインターフェース、モバイルアプリ、そして開発者向けの Claude Code（ターミナルで動作するアジェンティックコーディングツール）と API も影響を受けた。Anthropic はステータスページで障害を確認し、UTC 13:03 に修正・監視中と報告、同日中に解消された。
- **ユーザー反応**: ネガティブ（一時的）。「Claude Code が急に動かなくなって仕事が止まった」「APIが返ってこない」という開発者からの報告が X・Redditで急増した。解消後は「素早く直った」という評価に転じたが、「Claudeへの依存度が上がりすぎていると障害のダメージが大きい」という構造的な懸念も出た。
- **情報の確度**: 公式発表（Anthropic ステータスページ確認済み）＋ユーザー報告

### Google Flow — Whisk・ImageFX を統合した統一 AI クリエイティブスタジオに刷新

- **カテゴリ**: メジャーアップデート
- **AI領域**: 画像・動画・音声生成AI / デザインAI
- **概要**: Googleは2026年3月、AI動画・画像生成ツール「Flow」を大規模刷新し、これまで別々のサービスとして提供してきた「Whisk」（画像生成）と「ImageFX」（画像エフェクト）の機能を Flow の統一ワークスペースに統合した。ユーザーは3月から自分の Whisk・ImageFX プロジェクトと素材を Flow ライブラリへ移行するオプトインが可能になった。主な新機能として、Nano Banana モデル（Google の最新画像生成モデル）の組み込み、多シーン連続生成・自動トランジション生成、ImageFX キーフレーム注入による誘導生成、音声と映像の同期生成、Veo（Googleの動画生成モデル）とのシームレスな連携が追加された。これにより Flow は「動画生成ツール」から「静止画・動画・エフェクトを一括で扱うフルスタック AI クリエイティブパイプライン」へ進化した。
- **ユーザー反応**: ポジティブ。「3つのサービスを行き来する必要がなくなった」「Whiskで作った素材をそのまま動画に使えるフローが自然」という評価が多い。「競合の Adobe Firefly が同方向（統合スタジオ化）に向かっているのと同じトレンド」という指摘もある。
- **情報の確度**: 公式発表
- **ソース**: [Google Flow Just Merged Everything: What the March 2026 Redesign Means for AI Video Creators | Bonega.ai](https://bonega.ai/en/blog/google-flow-march-2026-unified-ai-video-workspace)（投稿日: 2026-03月、ソース: Bonega AI）、[Google rolls out updates to image and video tool Flow AI | Social Media Today](https://www.socialmediatoday.com/news/google-rolls-out-updates-to-image-and-video-tool-flow-ai/813166/)（ソース: Social Media Today）

### 領域横断の所見

3月17日の主な動向を俯瞰すると、「Claude エコシステムの強化と脆弱性」が同日に並んで表れた日だったことが際立つ。

一方では Claude Code の出力トークン上限拡張により、Anthropicがコーディング AI としての実用性を着実に高めている。他方、Claude AI の大規模障害は、ユーザー・開発者の Claude 依存度がすでに相当高いことを証明する形になった。「Claude Code なしで仕事が止まった」という声が多数出たことは、それだけ日常的なワークフローに組み込まれた証拠でもある。

また、Google が Flow に Whisk・ImageFX を統合したことは、AI クリエイティブ市場で「単機能ツールの乱立」から「統合スタジオの競争」への転換が加速していることを示す。Adobe Firefly も同方向に向かっており、2026年上半期の画像・動画生成 AI 市場は「ツール間の統合深度」が主要な競争軸になりつつある。
