## 用語ミニ解説

> 本レポートで使用する専門用語を冒頭で簡単に説明します。はじめてAI市場レポートを読む方はここを先に確認してください。

- **LLM（Large Language Model）**: 大規模言語モデル。大量のテキストで学習したAIで、文章の生成・要約・翻訳などができます（例: GPT-5、Claude、Gemini）。
- **AIエージェント**: 人間が細かく指示しなくても、自律的に複数のステップのタスクをこなせるAIシステムです。
- **MCP（Model Context Protocol）**: AIモデルが外部ツールやデータベースに接続するための標準規格。AIのための「共通コンセント」のようなものです。
- **オブザーバビリティ（Observability）**: システムの内部状態をログ・メトリクス・トレースなどから可視化・監視する仕組みを指します。DatadogはこのオブザーバビリティSaaSの代表格です。
- **RBAC（Role-Based Access Control）**: ロールベースのアクセス制御。ユーザーの役割に応じてシステムへのアクセス権限を管理する仕組みです。
- **GA（General Availability）**: 一般提供開始。ベータ版や限定プレビューから卒業し、全ユーザーが利用可能になった状態を指します。
- **Agent Skills**: GitHub Copilotに対して「指示書・スクリプト・リソースのフォルダ」を渡すことで、特定の専門知識やタスク実行能力を後付けで追加する仕組みです。

### Datadog — MCPサーバーGA、AIコーディングエージェントがリアルタイム本番監視データに直接アクセス可能に

- **カテゴリ**: 重要アップデート
- **AI領域**: AIエージェント・自動化 / AI開発基盤
- **概要**: Datadog（クラウド監視・オブザーバビリティSaaS）は2026年3月9日、MCPサーバーの一般提供（GA）を正式に発表した。これにより、Claude Code・Cursor・GitHub Copilot・OpenAI Codexなど主要なAIコーディングエージェントが、Datadogが収集するリアルタイムのログ・メトリクス・トレースデータに直接アクセスして本番障害を自律的にデバッグできるようになる。認証はユーザー単位のOAuth連携で行われ、既存のRBAC（ロールベースアクセス制御）ポリシーが自動的に適用されるため、ガバナンスを維持しながらAIエージェントへの権限付与が可能。「AIコパイロットから、本番システム上でAIが自律稼働するフェーズへの移行」をDatadogは明示的に謳っている。
- **ユーザー反応**: ポジティブ。X・Redditの開発者コミュニティでは「ついにAIエージェントがProduction Systemに直接触れる標準的な接点ができた」という評価が多数。一方で「権限の渡しすぎでインシデントが増えないか」というセキュリティ懸念の声も見られる。
- **情報の確度**: 公式発表
- **ソース**: [Datadog Launches MCP Server to Provide AI Agents with Secure, Real-Time Access to Unified Observability Data | Yahoo Finance](https://finance.yahoo.com/news/datadog-launches-mcp-server-ai-130000906.html)（投稿日: 2026-03-09、ソース: Datadog 公式プレスリリース / GlobeNewsWire）

- **ユーザー反応**: ポジティブ。「カスタム指示ファイルよりもさらに踏み込んだエージェントカスタマイズ手段が手に入った」という評価が多い。GitHubのCopilot Dev Days（3月15日から世界各地で開催予定）でのデモに期待する声も多数。
