## 用語ミニ解説

> 本レポートで使用する専門用語を冒頭で簡単に説明します。はじめてAI市場レポートを読む方はここを先に確認してください。

- **LLM**: 大規模言語モデル。大量のテキストで学習したAIで、文章の生成・要約・翻訳などができます（例: GPT-4、Claude、Gemini）。
- **AIエージェント**: 人間が細かく指示しなくても、自律的に複数のステップのタスクをこなせるAIシステムです。
- **MCP（Model Context Protocol）**: AIモデルが外部ツールやデータベースに接続するための標準規格。AIのための「共通コンセント」のようなものです。
- **コンテキストウィンドウ**: LLMが一度に処理できるテキストの量（トークン数）。大きいほど長い文書や会話を扱えます。
- **バイブコーディング（Vibe coding）**: AIに自然言語で意図を伝えながらコードを生成させる開発スタイルの俗称です。
- **ソースマップ**: バンドル・minify（圧縮）されたJavaScriptと元のソースコードの対応関係を記録したファイル。デバッグ用で、本番配布物には含めないのが通常です。
- **npmレジストリ**: Node.js/JavaScriptパッケージの公開配布プラットフォーム。開発者はここからライブラリをインストールします。
- **フィーチャーフラグ**: ソフトウェアの機能をコード上ではビルド済みだが、設定で有効/無効を切り替える仕組み。段階的リリースやA/Bテストに使われます。

### Claude Code — npmソースマップ経由で全ソースコード51万行超が流出

- **カテゴリ**: 実用トレンド / セキュリティインシデント
- **AI領域**: コーディングAI
- **概要**: 2026年3月31日、セキュリティ研究者のChaofan Shou氏がClaude Codeのnpmパッケージ（バージョン2.1.88）に59.8MBのソースマップファイルが含まれていることを発見し、X上で公表した。これにより約1,900ファイル・51万2,000行以上のTypeScriptソースコードが完全に復元可能な状態で露出。数時間以内にGitHub上に複数のミラーリポジトリが作成され、41,500回以上フォークされた。Anthropicはnpmパッケージを即座に更新しソースマップを除去、旧バージョンも削除したが、すでにコードは広範囲に拡散済みだった。
- **ユーザー反応**: 賛否両論 — エンジニアコミュニティでは「LLM企業が正規表現でユーザーの不満を検出しているのは皮肉」という反応が話題に。一方で、コードの品質やアーキテクチャ設計を高く評価する声も多く、技術的な分析記事が続々と投稿された。Anthropicにとっては、5日前のClaude Mythos流出に続く2回目のセキュリティインシデントであり、信頼性への懸念が拡大。
- **情報の確度**: 公式発表（Anthropicがリリースパッケージングの人的ミスと認めた）

### Claude Code — 流出コードから未発表の自律エージェント機能「KAIROS」が判明

- **カテゴリ**: 実用トレンド
- **AI領域**: AIエージェント・自動化
- **概要**: 流出したソースコードの中に、150箇所以上言及されるフィーチャーフラグ「KAIROS」が発見された。KAIROSは古代ギリシャ語で「好機」を意味し、Claude Codeを常時稼働のバックグラウンドデーモン（常駐プロセス）として動作させる自律エージェントモードである。15秒のブロッキングバジェットで能動的に行動を起こし、ユーザーがアイドル状態の間に「autoDream」と呼ばれるメモリ統合プロセスを実行する。研究プロトタイプではなく、本番コードベースにコンパイル済みのフィーチャーゲート付きコードとして存在しており、有効化を待つ段階にある。
- **ユーザー反応**: ポジティブ — 「これこそ次世代のAIコーディング体験」という期待の声が多数。既に類似のコンセプトを独自開発していた開発者からは驚きと競合懸念の声も。
- **情報の確度**: 信頼性の高い速報（流出ソースコードから直接確認されたが、Anthropicは公式にはこの機能について言及していない）

### Claude Code — 流出コードの主要な技術的発見事項まとめ

- **カテゴリ**: 実用トレンド
- **AI領域**: コーディングAI
- **概要**: 流出コードの分析により、Claude Codeの内部アーキテクチャが明らかになった。主な発見: (1) 約40のツールをプラグイン型アーキテクチャで管理し、各ツール（ファイル読み取り、Bash実行、Web取得、LSP統合等）がパーミッションゲート付きの独立モジュール。ベースツール定義だけで29,000行。(2) 46,000行のクエリエンジンがLLM APIコール、ストリーミング、キャッシュ、オーケストレーションを一元管理。(3) 44個のフィーチャーフラグで段階的リリースを管理。(4) 未発表の8機能・26の隠しコマンド・32のビルドフラグ・120以上のシークレット環境変数が存在。(5) 「Undercover Mode」はAnthropicの社員がOSSに貢献する際、AI生成の痕跡を隠す機能。(6) 「ANTI_DISTILLATION」フラグ有効時、偽のツール定義をシステムプロンプトに注入し、APIトラフィックを録画して競合モデルを学習させる行為を妨害。
- **ユーザー反応**: 賛否両論 — アーキテクチャ設計の品質に感嘆する声がある一方、「Undercover Mode」や反蒸留機能について倫理的懸念を示す意見も。Hacker Newsでは技術的な深掘り分析が活発に行われた。
- **情報の確度**: 信頼性の高い速報（複数の独立した分析者がソースコードを直接確認）

### 領域横断の所見

**Anthropicの連続リーク事件がAI業界全体に波及**: 3月26日のClaude Mythos（CMS設定ミス）に続き、3月31日のClaude Codeソースコード流出（npmパッケージングミス）と、わずか5日間で2回のセキュリティインシデントが発生。これは「AI企業の安全性」「プロプライエタリコードの保護」「npmサプライチェーンセキュリティ」という3つの異なる領域に横断的なインパクトを与えている。特にClaude Codeの流出からは、AIコーディングツールの内部アーキテクチャ（コーディングAI領域）、未発表の自律エージェント機能KAIROS（AIエージェント領域）、未発表のClaude Mythos/Capybaraモデルの参照（LLM領域）が同時に露出しており、単一のインシデントが複数のAI領域に連鎖的影響を及ぼした稀有な事例と言える。

**MCPの企業採用が加速**: Veeam、Domo、Google Colab、OpenAI Codexなど、異なる領域のプレイヤーがMCPサーバーを相次いでリリースしており、MCPがAIツール連携の事実上の標準として定着しつつある。
